-------------------------------------------------------------------

ebp ( Extended Base Pointer ) 		: 현 스택 프레임의 시작 주소
esp ( Extended Stack Pointer ) 		: 현 스택의 포인터
eax ( Extended Accumulator Register ) 	: 산술, 논리연산에 사용되는 레지스터

ecx ( Extended Counter Registr ) 	: 반복문 등의 카운팅을 위한 레지스터

ebx ( Extended Base Register ) 		: 메모리 주소를 저장하기 위한 용도 (반으로 쪼개면 BX(16bit))

esi ( Extended Source index ) 		: 데이터 복사시 원본 주소를 말함

edi (Extended Destination index) 	: 데이터 복사시 목적지 주소

-------------------------------------------------------------------




전역함수가 성능이 떨어지는 이유

레지스터에는 세그먼트 공간이 따로 존재한다.(데이터 영역)
전역변수는 세그먼트를 기준으로 상대주소로 찾아간다.
스택은 epb를 기준으로 상대주소값으로 계산한다.

접근자체에 성능이 떨어질 이유는없으나,
캐시히트율이 떨어질 가능성이 높아진다.

-----------------------------------------------------

문자열의 줄바꿈 표현 (\n)

windows 	0x 0D 0A

C언어 		0x 0A 


텍스트파일에서 1A는 파일의 끝.


바이너리 파일을 텍스트파일로 열게되는경우,
0D 데이터가 날아가고 1A를 만나 파일이 끝나게된다.

모든파일을 바이너리로 사용한다고 하더라도 문제는없다.

텍스트에디터를 사용할경우 이같은 상황을 자동으로 해결해준다.

-----------------------------------------------------

텍스트 에디터를 하나쯤 익혀야 하는 이유


서버운영시 텍스트로 에러로그를 남길때가있는데,
이때 메모장으로 열고자한다면 메모리할당시 문제가 날 가능성이 있기때문에 에디터를 사용한다.

파일 패킹

1. 패킹 헤더

[파일 개수, 내꺼맞는지 코드정도]


2. 파일 헤더

[파일 이름, 확장자명, 오프셋]


3. 들어가는 모양새

[패킹헤더][파일헤더]*n] 데이터 데이터 데이터...

티어링 현상

60Hz		->	초당 60번 주사

200프레임	->	1초에 200번그린다

1프레임당 0.001 m/s 

아직 주사도 다못했는데 Flip하는경우, 화면이 찢기는듯한 현상을 말함



-----------------------------------------------------

티어링 현상이 있어도 동기화를 가지않는이유?

동기화프레임이 어긋나는 경우 프레임이 반토막이 남.
Filp해야 하는데 넘기지못했다면 한번더 기다려야함


현재 윈도우 환경에서 인터럽트 신호를 직접 받아내는 방법은 없다.
하드웨어적인 접근을 윈도우에서 모두막았음.

-----------------------------------------------------

bmp파일

bmp파일가장 상단에는 헤더가 존재한다.

파일헤더(BITMAPFILEHEADER) + 그림헤더(BITMAPINFOHEADER)

bmp파일같은 경우는 좌우는 그대로이나 위아래가 뒤집혀있다.


-----------------------------------------------------

RGB

RGB(0,0,0) = 검정
RGB(255,255,255) = 흰색


-----------------------------------------------------

CMYK

RGB와 반대되는방식.
Cyan(파랑), Magenta(자주), Yellow(노랑), Key(검정)의 배합으로 색표현.
모두 섞으면 검정이된다.


-----------------------------------------------------


이미지 합성

각 RGB요소를 더하여 평균값을 내는 방식으로 블렌딩한다.
합성이전에 픽셀안에 색순서가 ARGB인지, BGRA인지 알아야한다.


-----------------------------------------------------

이미지 흑백화

RGB가 완전히 같다면 밝고 어두운것만 표현이 가능하다.
RGB를 모두더해서 평균값을 낸다음 각자리에 들어거면 흑백으로표현됨
